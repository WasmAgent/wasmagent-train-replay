# Agent Tool Integration

This page shows how an LLM agent calls the `wasmagent-train-replay` tool
interface from a `tool_use` message, using JSON Schema definitions that match
the Milestone 4 agent tools.

The examples assume a training trace has already been dumped to
`artifacts/runs/demo/nccl_trace.pkl` and that tensor `tensor:2:3:out` was
identified as anomalous.

## Tool Definitions

Expose the tools to the agent as JSON Schema described functions:

```json
[
  {
    "name": "trace_tensor",
    "description": "Trace an anomalous tensor entity to the collective activities that causally contributed to it.",
    "input_schema": {
      "type": "object",
      "additionalProperties": false,
      "required": ["entity_id", "dump_path"],
      "properties": {
        "entity_id": {
          "type": "string",
          "description": "PROV entity id, for example tensor:2:3:out."
        },
        "dump_path": {
          "type": "string",
          "description": "Path to a Flight Recorder dump readable by wasmagent-train-replay."
        }
      }
    }
  },
  {
    "name": "list_suspicious_actions",
    "description": "List FULL-mode AEP records for a run and epoch.",
    "input_schema": {
      "type": "object",
      "additionalProperties": false,
      "required": ["dump_path", "run_id", "epoch"],
      "properties": {
        "dump_path": {
          "type": "string",
          "description": "Path to the evidence or trace dump."
        },
        "run_id": {
          "type": "string",
          "description": "Training run identifier."
        },
        "epoch": {
          "type": "integer",
          "minimum": 0,
          "description": "Epoch number to inspect."
        }
      }
    }
  },
  {
    "name": "summarize_epoch",
    "description": "Return aggregate evidence statistics for a run and epoch.",
    "input_schema": {
      "type": "object",
      "additionalProperties": false,
      "required": ["dump_path", "run_id", "epoch"],
      "properties": {
        "dump_path": {
          "type": "string",
          "description": "Path to the evidence or trace dump."
        },
        "run_id": {
          "type": "string",
          "description": "Training run identifier."
        },
        "epoch": {
          "type": "integer",
          "minimum": 0,
          "description": "Epoch number to summarize."
        }
      }
    }
  }
]
```

## Worked `tool_use` Message

When the agent sees a bad tensor on rank 2 at sequence 3, it should call
`trace_tensor` with the tensor entity id and the trace dump path:

```json
{
  "type": "tool_use",
  "id": "toolu_trace_tensor_001",
  "name": "trace_tensor",
  "input": {
    "entity_id": "tensor:2:3:out",
    "dump_path": "artifacts/runs/demo/nccl_trace.pkl"
  }
}
```

The tool implementation builds a provenance graph from the dump and runs:

```python
EpochReplayer(graph).find_root_cause("tensor:2:3:out")
```

## Sample `tool_result`

Return structured JSON so the agent can quote the root-cause path, inspect the
rank and sequence involved, and decide whether to call another tool such as
`list_suspicious_actions`.

```json
{
  "type": "tool_result",
  "tool_use_id": "toolu_trace_tensor_001",
  "content": [
    {
      "type": "json",
      "json": {
        "tool": "trace_tensor",
        "entity_id": "tensor:2:3:out",
        "dump_path": "artifacts/runs/demo/nccl_trace.pkl",
        "root_cause": {
          "causal_ancestors": ["act:2:all_reduce:3"],
          "primary_activity": "act:2:all_reduce:3",
          "rank": 2,
          "sequence_id": 3,
          "collective_type": "all_reduce"
        },
        "summary": "tensor:2:3:out was generated by rank 2 all_reduce sequence 3; inspect FULL-mode AEP records for that rank and epoch next."
      }
    }
  ],
  "is_error": false
}
```

The important contract is that `causal_ancestors` contains activity ids returned
by `EpochReplayer.find_root_cause(entity_id)`. For tensors created by
`build_from_events()`, ids follow the conventions documented in
`docs/cli-reference.md`: input tensors are `tensor:{rank}:{sequence_id}:in`,
output tensors are `tensor:{rank}:{sequence_id}:out`, and collective activities
are `act:{rank}:{collective_type}:{sequence_id}`.
